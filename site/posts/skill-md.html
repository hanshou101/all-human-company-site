SKILL.MD

为什么一个链接就能让 Agent 学会使用网站

UI 是给人类点的。技能说明书是给 Agent 读的：它告诉你“可以做什么、怎么做、边界在哪里”。

核心要点

  • 可发现：固定 URL（例如 /skill.md），方便 Agent 直接加载。
  • 可执行：不仅是宣言，要给出明确 endpoint / payload / 示例。
  • 可约束：写清楚“允许做什么、不允许做什么”，防止误用。

为什么它像 API 文档

Agent 不是来“浏览”的，它是来“完成任务”的。skill.md 的价值在于把可调用能力封装成一组稳定的动作。

全人公司的写法

我们在 /skill.md 用 YAML frontmatter 提供最低限度的元信息（name/description/version）， 然后在正文写 MCP 的最小调用路径。

真实世界里，很多生态就是这么干的（OpenClaw/Moltbook 等）。

一句话模板

“Read pages for context → call tools for action → keep payload small → bring coffee.”
